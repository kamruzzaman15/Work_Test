## Dataset

All sequencing data comes from SecureBio's **Zephyr** program
([data.securebio.org/zephyr](https://data.securebio.org/zephyr)): pooled nasal-swab
metatranscriptomic sequencing (Oxford Nanopore long reads) from Boston-area collection sites,
already filtered to vertebrate-infecting-virus reads and scrubbed of human genetic material
before publication. Each downloadable file is one **swab pool** — reads from several
individuals' swabs physically combined into one sequencing run, not one person.

For Parts 2 and 3 we used **12 swab pools** spanning **8 distinct collection sites** and dates
from 2026-03-18 to 2026-06-13 (~490,000 reads total, ~3.2kb average read length). Pool
selection and download: [`part2_taxonomic_classification/scripts/part2_step1_download_pools.sh`](part2_taxonomic_classification/scripts/part2_step1_download_pools.sh).

## Setup

One shared conda environment (`biosurv`) covers Parts 2 and 3:

```bash
conda env create -f part2_taxonomic_classification/environment.yml
conda activate biosurv
```

This installs: `kraken2`, `bracken`, `minimap2`, `samtools`, `seqkit`, `entrez-direct`,
`sourmash`, plus `python`/`pandas`/`matplotlib`/`scikit-learn`/`scipy`.

Part 2 additionally needs the prebuilt Kraken2 "Viral" RefSeq database (~1GB, not checked
into the repo):
```bash
wget https://genome-idx.s3.amazonaws.com/kraken/k2_viral_20240904.tar.gz
mkdir -p part2_taxonomic_classification/data/kraken2_db
tar -xzf k2_viral_20240904.tar.gz -C part2_taxonomic_classification/data/kraken2_db
```

Large/regenerable files (raw pool FASTAs, the Kraken2 database, BAMs, per-read depth files)
are excluded via `.gitignore` and regenerated by running the numbered scripts in order — see
each part's `*_RESULTS.md` for the exact commands.

## Part 2 — Taxonomic classification

**Question answered:** what viruses are present across the 12 pools, how confident are we,
and what genome coverage do we see for common respiratory viruses?

**Tools/models used** (all existing, off-the-shelf — nothing trained or fine-tuned):
- **Kraken2** (k-mer-based taxonomic classifier) with the prebuilt **NCBI Viral RefSeq**
  database, plus **Bracken** for species-level abundance re-estimation
- **minimap2** (`map-ont` preset, built for Nanopore long reads) to align reads against
  **NCBI RefSeq reference genomes** (fetched live via `efetch`) for genome coverage
- **samtools** for depth/breadth statistics

**Headline result:** Rhinovirus (species A/B/C) dominates almost every pool with
90-100% genome breadth where present; RSV, SARS-CoV-2, human metapneumovirus, parainfluenza,
Influenza C, and two seasonal coronaviruses (NL63, 229E) appear at much lower abundance with
partial, patchy genome coverage. Full details, caveats, and a key methodological finding
(Kraken2's default confidence filter is miscalibrated for long ONT reads) are in
[`part2_taxonomic_classification/part2_RESULTS.md`](part2_taxonomic_classification/part2_RESULTS.md).

## Part 3, Track B — Embedding-based clustering

**Question answered:** embed sequences and cluster them — do the clusters match the
taxonomy profile (from Part 2) or ANI?

**Tools/models used:**
- **sourmash** (MinHash k-mer sketching) for the sequence embedding/similarity — chosen
  specifically to avoid downloading any pretrained protein/nucleotide language model; a
  sourmash similarity score is itself a standard ANI estimator, so one tool covers both
  halves of the question
- **DBSCAN** + classical **MDS** (scikit-learn) for clustering and 2D visualization
- Ground-truth taxonomy reused directly from Part 2's Kraken2 calls — no new classification

**Headline result:** Adjusted Rand Index of 0.56 between unsupervised clusters and Part 2's
taxonomy labels, and within-species nucleotide similarity ~47x higher than between-species
similarity — strong evidence the embedding captures real taxonomic/ANI structure. Full
details, including a clustering-method pitfall we caught and fixed along the way, are in
[`part3_embedding_clustering/part3_RESULTS.md`](part3_embedding_clustering/part3_RESULTS.md).
